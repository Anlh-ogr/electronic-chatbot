PHASE 0 - KHỞI TẠO HỆ THỐNG (TUẦN 1)
Mục tiêu:
- Chuẩn hóa cấu trúc project electronic-chatbot.
- Có data JSON circuit_scope.json làm "bộ não" mạch cơ bản (4 mạch).
Nội dung:
- Tạo tree folder như hiện tại (apps/api, tools, docs, web...).
- Viết convert_yaml_to_json.py để build circuit_scope.json từ circuit_scope.yaml.
- Viết CircuitStore (circuit_store.py) để load/query data.
- Viết matcher rule-based (matcher.py) và test_matcher.py để kiểm tra keywords + priority.
Kết quả đạt được:
- Chạy script.py và test_matcher.py thành công, match đúng 4 mạch: buck LM2596, boost MT3608, op-amp inverting LM358, NE555 music.
- Có circuit_scope.json commit vào git làm database Phase 1.
Tồn đọng:
- Chưa có API /chat.
- Chưa có formatter chuẩn hoá câu trả lời.
- Chưa có UI/web.

PHASE 1 - CHATBOT RULE-BASED CỐT LÕI (TUẦN 2–4)
Mục tiêu:
- Người dùng nhập text → hệ thống nhận dạng đúng mạch cơ bản → trả lời chuẩn hóa (không AI).
- Hình thành "hợp đồng API" và schema dữ liệu để về sau dễ thay DB/AI.
Nội dung:
- Viết models/schemas.py (ChatRequest, ChatResponse).
- Viết services/formatter.py: render Nguyên lý / Công thức / BOM / Giải thích dựa trên specs + knowledge cơ bản.
- Viết api/routes/chat.py: /api/chat nhận message, dùng CircuitStore + matcher + formatter để trả response.
- Hoàn thiện app/main.py: tạo FastAPI app, load CircuitStore, include router /api.
- Bổ sung test thủ công qua Swagger UI.
Kết quả đạt được:
- POST /api/chat hoạt động, match đúng 4 mạch theo keywords + priority.
- Trả lời ổn định với format thống nhất (text), có fallback_response khi không hiểu.
Tồn đọng:
- Chưa parse Vin/Vout/Iout từ câu người dùng để check với specs.
- Chưa có knowledge chi tiết (BOM cụ thể, công thức đầy đủ, hình ảnh...).
- Chưa tách Data Access Layer (CircuitRepo) để thay DB.

PHASE 2 - MỞ RỘNG KIẾN THỨC & HẠ TẦNG DỮ LIỆU (TUẦN 5–8)
Mục tiêu:
- Biến circuit_scope.json thành cơ sở kiến thức phong phú cho 4 mạch.
- Chuẩn hoá cách lưu knowledge để sau chuyển sang PostgreSQL/jsonb dễ dàng.
Nội dung:
- Mở rộng schema JSON:
  - Thêm field knowledge cho từng circuit: principle_md, formulas[], bom[], notes[], images[], simulation, eda.
  - Viết lại convert_yaml_to_json.py để build đúng schema mới.
- Cập nhật formatter để:
  - Đọc các field knowledge (nếu có) và render đẹp.
  - Giữ fallback khi field chưa có.
- Bắt đầu tách Data Layer:
  - Định nghĩa interface CircuitRepo (list_circuits, get_by_id, search_by_keywords).
  - Implement JsonCircuitRepo dùng circuit_scope.json.
- Bổ sung unit test (pytest) cho matcher + formatter + repo.
Kết quả đạt được:
- Data cho 4 mạch có: nguyên lý, công thức cơ bản, BOM gợi ý, ghi chú.
- Code backend chỉ dùng CircuitRepo thay vì đọc file trực tiếp.
Tồn đọng:
- CSDL vẫn là file JSON, chưa có CRUD/phiên bản hoá.
- Chưa có AI, chưa có RAG.

PHASE 3 - KIẾN TRÚC HỆ THỐNG & CHUẨN BỊ AI (TUẦN 9–12)
Mục tiêu:
- Thiết kế sơ đồ khối hệ thống chatbot AI: từ Web/App → API → AI → EDA.
- Chuẩn bị chỗ gắn AI (ChatGPT/Colab) mà không phá core rule-based.
Nội dung:
- Vẽ sơ đồ khối:
  - Web/App UI
  - FastAPI Gateway
  - Circuit Service (Repo + knowledge)
  - Match Service (rule-based)
  - AI Service (gọi API AI / Google Colab)
  - EDA Service (sinh schematic/PCB)
  - Data: JSON/PG + file storage.
- Trong code:
  - Tạo module ai_client (stub): AIClient.analyze(message) → {intent, params, explanation_md}.
  - Orchestrator trong /chat: giữ flow đơn giản, nhưng tách rõ phần có thể gọi AI sau.
- Viết tài liệu nội bộ: data flow từ input → matcher → circuit → formatter → (tương lai) AI/EDA.
Kết quả đạt được:
- Kiến trúc rõ ràng, code phân lớp (API, services, repo, ai, eda).
- Sẵn sàng thay backend dữ liệu (Postgres) hoặc gắn AI mà không sửa nhiều route.
Tồn đọng:
- AI chưa thực sự được tích hợp.
- Chưa có cơ sở dữ liệu cloud.

PHASE 4 - CƠ SỞ DỮ LIỆU CLOUD (POSTGRESQL/JSONB) & MỞ RỘNG MẠCH (TUẦN 13–16)
Mục tiêu:
- Đưa dữ liệu mạch lên PostgreSQL (cloud/local), dùng jsonb cho specs/knowledge.
- Cho phép mở rộng thêm các mạch cơ bản khác ngoài 4 mạch hiện tại.
Nội dung:
- Thiết kế schema PostgreSQL:
  - circuits(id, name, category, ic, specs jsonb, knowledge jsonb, created_at, updated_at).
  - keywords(circuit_id, keyword).
  - (sau) projects, sim_files, eda_files.
- Viết PostgresCircuitRepo (dùng SQLAlchemy/SQLModel) implement CircuitRepo.
- Seed DB từ circuit_scope.json.
- Thêm vài mạch cơ bản mới (ví dụ: linear regulator, LDO, đơn giản khác) bằng insert dữ liệu.
- Cho phép chọn backend qua config: DATA_BACKEND=json|postgres.
Kết quả đạt được:
- Backend có thể chạy với JSON (dev) hoặc Postgres (cloud) mà API `/chat` không đổi.
- Thêm mạch mới chỉ cần thêm record vào DB, không sửa matcher/formatter.
Tồn đọng:
- Chưa có AI hiểu ngôn ngữ tự nhiên sâu.
- Chưa có mô phỏng mạch.
- Chưa sinh được schematic/PCB.

PHASE 5 - WEB/APP UI & AI CHATBOT (TUẦN 17–18)
Mục tiêu:
- Có giao diện Web/App trên PC để chat với chatbot.
- Bắt đầu dùng AI (ChatGPT/Colab) để hỗ trợ hiểu yêu cầu và diễn giải, nhưng vẫn bị giới hạn bởi data.
Nội dung:
- Frontend (React/Next):
  - Màn hình chat.
  - Panel hiển thị mạch được chọn, specs, BOM, notes.
  - Nút "Generate schematic" / "Simulate" (Phase 6 mới hoạt động đầy đủ).
- Backend AI integration:
  - AIClient gọi API AI hoặc Google Colab.
  - Flow: user message → AIClient (trích intent & thông số) → matcher/DB → formatter → (AI paraphrase nếu cần).
  - Bảo toàn guardrail: chỉ cho phép mạch trong DB, out_of_scope vẫn do rule-engine chặn.
Kết quả đạt được:
- Người dùng có thể chat trên web/app, chatbot trả lời tự nhiên hơn, nhưng mạch vẫn trong phạm vi an toàn.
Tồn đọng:
- Chưa sinh schematic KiCad.
- Chưa mô phỏng và PCB.

PHASE 6 - EDA: SCHEMATIC → PCB + MÔ PHỎNG (TUẦN 19–20)
Mục tiêu:
- Từ yêu cầu → chọn mạch → sinh schematic KiCad.
- Từ schematic → hỗ trợ tạo PCB (ít nhất là layout sơ bộ, vị trí XY linh kiện).
- Tích hợp workflow mô phỏng (PSIM/PCBSim hoặc tool miễn phí tương đương).
Nội dung:
- Schematic (KiCad):
  - Chuẩn bị file template `.kicad_sch` cho từng mạch (buck LM2596, boost MT3608, op-amp LM358, 555 music).
  - Trong knowledge.eda của mỗi circuit lưu: tên template, mapping refdes, giá trị linh kiện, vị trí khối.
  - Backend EDA Service:
    - Copy template schematic.
    - Sửa giá trị R/L/C, text theo bom + thông số user.
    - Xuất file `.kicad_sch` cho người dùng tải.
- PCB layout:
  - Chuẩn bị template `.kicad_pcb` với footprint & net đã mapping cho từng mạch cơ bản.
  - Lưu trong knowledge.eda thông tin placement_hint (x,y) cho từng group linh kiện.
  - Backend chỉnh toạ độ XY cho linh kiện trong file `.kicad_pcb` theo rule đơn giản (power stage gần nhau, input/output edge, v.v.).
  - Xuất PCB cho user mở trong KiCad, không cần sinh 3D.
- Mô phỏng:
  - Chuẩn bị mẫu netlist mô phỏng (PSIM/SPICE/PCBSim) cho 4 mạch.
  - Lưu path file mô phỏng trong knowledge.simulation.
  - Backend sinh file mô phỏng tương ứng (thay đổi Vin/Vout/Rload nếu có thể) và cung cấp cho user chạy bằng PSIM/PCBSim hoặc tool miễn phí.
Kết quả đạt được:
- Chatbot AI có thể: hiểu yêu cầu → chọn mạch → xuất schematic KiCad → xuất PCB layout cơ bản.
- Người dùng có thể tải file về, mở bằng KiCad và PSIM/PCBSim để xem/simulate.
Tồn đọng:
- Tối ưu tự động layout PCB còn cơ bản; chưa auto-route phức tạp.
- Chưa hỗ trợ nhiều mạch phức tạp hơn.

PHASE 7 - MỞ RỘNG MẠCH & TỐI ƯU HỆ THỐNG (PHẦN CÒN LẠI CỦA 5 THÁNG)
Mục tiêu:
- Mở rộng từ 4 mạch sang nhiều mạch cơ bản khác.
- Cải thiện chất lượng AI, mô phỏng, và hỗ trợ PCB.
Nội dung:
- Thêm mạch cơ bản mới (LDO, nguồn tuyến tính, khuếch đại không đảo, lọc RC, mạch logic đơn giản,...).
- Cho mỗi mạch mới:
  - Thêm record trong DB (Postgres), gồm specs, knowledge, keywords, eda, simulation.
  - Tạo/nhúng template schematic/PCB (nếu dùng cách "vẽ sẵn và lưu trong DB").
- Cải thiện AI:
  - Use RAG: embed knowledge (principle, notes, bom) và cho AI truy xuất khi trả lời.
  - Logging & feedback: thu thập câu hỏi thực tế và điều chỉnh keyword/knowledge.
- Tối ưu EDA:
  - Cải thiện quy tắc placement XY.
  - Nghiên cứu script KiCad để hỗ trợ update schematic → PCB mượt hơn.
Kết quả đạt được:
- Hệ thống chatbot AI hỗ trợ thiết kế mạch điện tử cơ bản theo ứng dụng, từ chat → schematic → PCB ở mức demo thực tế.
- Dữ liệu và kiến trúc đủ linh hoạt để tiếp tục mở rộng sau 5 tháng.
